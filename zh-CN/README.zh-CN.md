# R 临床预测模型 Codex Skill：安装与使用指南

[English version] 见英文版文件夹或主页仓库README

一个面向医学研究的双语 Codex skill，用于在 R 中构建、验证和报告可复现的临床预测模型。它覆盖数据契约、数据导入与类型检查、Logistic 回归、Cox 比例风险模型、竞争风险、校准、判别、临床效用以及透明、可复核的结果报告。

> 本仓库只包含 Codex skill 本身和教学资料，不包含患者数据、研究结果、教材 PDF 或教材扫描件。本项目不提供个体医疗建议；未经充分的外部验证、校准、临床效用评估和治理审查，不应宣称模型可用于临床部署。

## 项目特点

- **减少重复编码：** 将临床预测建模中的通用步骤整理为可复用工作流。原始教学资料中“节约 70% 时间”是作者的个人使用体验，不是对所有任务的性能保证。
- **强调数据契约：** 在拟合模型前明确变量含义、类型、编码、缺失值、时间单位、结局定义和纳排标准。
- **覆盖常见模型：** 支持 Logistic、Cox 和竞争风险等研究级分析流程。
- **重视验证与报告：** 同时关注比例风险假设、校准、判别、临床效用、内部或外部验证及透明报告。
- **可复现：** 可以要求输出带中文注释的 R 脚本、处理日志、中间结果、表格和矢量图。
- **双语版本：** 中文和英文 skill 可以按需求择一安装。

## 安装

### 方法一：让 Codex 协助安装

可以把下面的提示词发给 Codex；将仓库地址替换为实际 GitHub 地址：

```text
请从以下 GitHub 仓库下载 clinical-prediction-modeling-r-skill：
https://github.com/yangkaikai/clinical-prediction-modeling-r-skill

请将中文版本 zh-CN/clinical-prediction-modeling-r 安装到我的 Codex skills 目录，
检查 SKILL.md 和所需文件是否完整，并告诉我安装位置和验证结果。
```

### 方法二：手动安装中文版本

```bash
git clone https://github.com/yangkaikai/clinical-prediction-modeling-r-skill.git
mkdir -p ~/.codex/skills
cp -R clinical-prediction-modeling-r-skill/zh-CN/clinical-prediction-modeling-r ~/.codex/skills/
```

安装英文版本时，把复制路径中的 `zh-CN` 改为 `en-US`。两个版本的安装目录名相同，因此同一位置只能保留其中一个；需要切换语言时请先备份或替换现有版本。

> 如果实际 GitHub 用户名或仓库名不同，请同步替换命令中的 URL。

## 使用前需要准备什么

### 1. 数据说明或数据字典

建议准备 Markdown、Excel 或 CSV 格式的数据说明，至少包含：

- 列名和中文或英文含义；
- 数据类型：连续型、名义分类、有序分类、日期时间或时间依赖变量；
- 单位、取值范围和分类编码；
- 缺失值、`unknown`、拒答或特殊编码的含义；
- 暴露、结局、随访时间、事件状态和协变量的定义；
- 预先规定的纳入、排除与数据清洗规则。

### 2. 数据文件

可以直接上传，或提供本地绝对路径。医学数据建议限定在本地工作目录中读取和输出，不要上传患者身份信息、未充分去标识化的数据、访问密钥或未经授权的资料。

### 3. 明确的分析需求

至少说明：

- 研究问题和目标模型；
- 暴露、结局、随访时间和候选协变量；
- 单因素、多因素、非线性、交互或敏感性分析要求；
- 缺失数据策略和变量类型处理要求；
- 需要的表格、图形、文件格式和保存目录；
- 是否需要 R 脚本、工作日志、数据处理说明和质量检查报告。

### 4. 目标表格或图形

如果期望特定版式，可以上传示例图或表，并明确哪些元素必须复刻，例如列标题、参考组、颜色、字体、坐标轴、置信区间和脚注。

## Cox 分析提示词示例

下面的模板适合暴露四分位分组、单因素和全协变量多因素 Cox 分析。请根据自己的数据替换方括号内容。

```text
使用 $clinical-prediction-modeling-r。

请在本地工作目录内完成 Cox 比例风险模型分析。

数据文件：[数据文件路径]
变量说明：[数据字典路径]
输出目录：[输出目录]
目标森林图示例：[图片路径，可选]

研究定义：
- 随访时间变量：[变量名及单位]
- 事件状态变量：[变量名；0/1 的含义]
- 暴露变量：[暴露1、暴露2……]
- 候选协变量：[变量列表；若要求全部纳入，请明确写出]
- 排除规则：[预先规定的规则]
- 缺失值规则：[完整病例、多重插补或其他预先规定策略]

分析要求：
1. 先核对数据字典，并明确每个变量是连续型、名义分类、有序分类还是时间依赖变量。
2. 每个连续暴露分别按分析样本中的四分位数切点分为 Q1、Q2、Q3、Q4，Q1 为参考组；报告切点、各组样本数和事件数，并处理重复切点问题。
3. 对每个暴露分别建立单因素 Cox 模型。
4. 对每个暴露分别建立多因素 Cox 模型，并纳入我指定的全部协变量；不要依据单因素 P 值筛选变量。
5. 报告 HR、95% CI、P 值和 P for trend，并说明趋势检验的编码方法。
6. 检验比例风险假设，报告全局检验和变量层面的结果；若违反假设，不要静默忽略，应报告并提出预先可解释的处理方案。
7. 仅针对多因素模型绘制 Q1–Q4 的调整后生存曲线或明确说明所采用的标准化方法；不要把未经调整的 KM 曲线误称为调整后 KM 曲线。
8. 分别生成单因素和多因素森林图，版式尽量匹配示例图；同时提供可编辑的结果表。

交付文件：
- 带中文注释的完整 R 脚本；
- 数据清洗和每一列变量处理说明；
- 单因素与多因素结果 CSV/XLSX；
- 森林图和生存曲线 PDF 矢量图；
- 比例风险假设诊断图；
- 分析说明与工作日志 Markdown；
- sessionInfo() 或等价的运行环境记录。

请先输出数据审计和建模计划；发现变量定义、时间零点、事件编码或协变量处理存在重大歧义时，暂停并询问我。
```

## 建议工作流程

1. **先询问能力边界：** 让 Codex 说明该 skill 能做什么、不能做什么。
2. **明确输入与输出：** 指定数据、数据字典、示例图、工作目录和交付文件。
3. **先审计后建模：** 先确认变量类型、编码、缺失数据、纳排规则和时间定义。
4. **补充复现要求：** 要求 R 脚本、运行环境、随机种子、处理日志和中间结果。
5. **人工复核：** 检查样本量、事件数、模型公式、参考组、HR/CI、假设检验和图表是否一致，再决定是否用于研究报告。

## 复现、安全与结果核验

- 对输入文件使用只读方式，结果输出到独立目录。
- 不要提交患者数据、访问凭据、教材 PDF、内部路径或受限资料到公开仓库。
- 记录原始样本量、排除人数、最终分析样本量和事件数。
- 记录每个变量的清洗、重编码、参考组和缺失数据处理。
- 固定随机种子，并保存 R 版本、包版本和 `sessionInfo()`。
- 对关键结果进行独立复核；自动化不能替代统计学判断、临床解释和研究治理。
- 森林图、表格和正文中的样本量、HR、95% CI 与 P 值必须相互一致。
- 不应仅依据 P 值选择协变量；调整集应优先基于研究问题、因果结构和预先规定的分析方案。
